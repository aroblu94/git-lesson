Git

Cos'è

Git è un sistema di controllo versione (VCS) e serve quindi a registrare i cambiamenti che vengono effettuati su un file o su una serie di file nel tempo, così da poter richiamare una versione specifica di quei dati in qualsiasi momento e tenerne sotto controllo ogni modifica.

Software come Git possono essere utilizzati in locale per tracciare i progressi di un progetto personale, così come online per gestire un progetto di gruppo.

Grazie a un VCS si può:

  • ripristinare i file ad uno stato precedente
  • ripristinare l’intero progetto allo stato precedente
  • confrontare i cambiamenti all’interno dei file
  • nel caso di lavoro in team, vedere chi ha realizzato determinate modifiche, quando e perché
  • ...

Git è un VCS distribuito. Tutti gli utenti che lavorano al progetto hanno infatti una copia completa del repository (l’insieme di tutte le versioni dei vari file). In questo modo è come se esistesse una copia di backup per ogni persona che lavora al progetto, senza che i membri del team abbiano bisogno di connettersi al server per visualizzare la storia di un progetto.

Come lavora

Repository

È il nome dato ad ogni progetto Git. Per creare una repository si può partire da un progetto non su Git oppure clonare una repository già esistente.

Branch

Un progetto controllato da Git può avere diverse diramazioni, o branch, vale a dire dei flussi di sviluppo differenti per ogni progetto. Questo permette di lavorare allo sviluppo (di una nuova feature, della correzione di un bug, ...) senza correre il rischio di pasticciare il ramo principale del progetto.
Una volta raggiunto l'obiettivo del dato branch, si può aggiornare il ramo principale (master) effettuando un merge tra i due rami.

Gestione dei file

I file controllati da Git (tracked) possono assumere tre stati diversi:

  • unmodified - il file non è stato modificato
  • modified - il file è stato modificato ma non è stato contrassegnato in modo da essere aggiunto al database
  • staged - il file modificato è stato contrassegnato in modo da essere inserito nel database

Commit

Un commit è il "componente base" di Git e registra uno stato dello sviluppo di un progetto, riportando diverse informazioni sulle ultime modifiche rispetto al commit precedente:

  • a quale ramo - branch - abbiamo affidato il commit
  • checksum (versione) del commit
  • autore del commit
  • timestamp del commit
  • descrizione e commenti al commit inseriti dall'autore
  • quanti e quali file sono stati modificati
  • statistiche sulle linee aggiunte, eliminate o modificate nel commit

Qualche comando

git init - inizializza una repository Git
git clone [url] [destinazione] - clona una repository Git
git status - mostra lo status della repository
git add [file] - traccia un file
git rm [file] - rimuove un file da quelli tracciati
git commit [-m "commento"] - crea un commit con le modifiche effettuate e tracciate
git pull [repository] [branch] - sincronizza la repository locale con la versione su server
git push [repository] [branch] - aggiorna la versione della repository sul server con le modifiche locali committate

GitHub

GitHub è una piattaforma di web hosting nata per ospitare progetti Git. Permette di gestire le proprie repository online in modo semplice e di condividere i progetti con altri sviluppatori.
Chiunque può visualizzare il codice dei repository pubblici e mettersi in contatto con lo sviluppatore del progetto stesso, può effettuarne un fork, vale a dire importarne una copia sul suo account, clonarlo e contribuirvi.
GitHub offre per ogni repository un sistema di bug tracking che permette di tenere traccia di eventuali bug e possibili enhancement ad ognuno dei quali è possibile aggiungere commenti o discussioni tra sviluppatori.
Questo permette ad ogni sviluppatore, anche se non fa parte del team di sviluppo del progetto in questione, di aiutare il team nello sviluppo del progetto, attraverso i meccanismi di fork, clone, merge e pull request.
